퀵스타트¶

보케는 최신 웹 브라우저에서 표현하는 것을 목표로하는 상호작용형 시각화 라이브러리다. 보케는 대규모 혹은 스트리밍 데이터셋에 대해 파이썬 (또는 다른 언어로) 빠르고 쉽게 다양한 그래픽을 간결 정확하게 만드는 것을 목표로 한다.

단순함과 강력함, 그리고 고급 커스터마이징을 위한 유연성을 제공하기 위해 보케는 사용자에게 두가지 인터페이스 레벨을 제공한다.:

bokeh.models
애플리케이션 개발자에게 가장 유연한 환경을 제공하는 로우레벨 인터페이스
bokeh.plotting
시각적 글리프를 조립하기 위한 하이레벨 인터페이스

이 퀵스타트는 bokeh.plotting 인터페이스에 중점을 둔다.

설치¶

보케를 설치하는 방법은 여러가지이지만 가장 쉬운 방법인 Anaconda Python distribution 사용을 추천한다. 배시나 윈도우 명령 프롬프트에서 다음 명령을 실행한다.:

conda install bokeh

이렇게 하면 보케가 필요로 하는 모든 의존 라이브러리도 설치되낟. 아나콘다를 쓰면 윈도우를 포함하여 모든 플랫폼과 설정에서 설치가 가장 쉽다. 또한 examples/ 서브디렉토리에 예제들도 설치된다.

만약 NumPy 등 모든 필요 퍄키지 설치가 되었다면 pip 명령을 써도 된다.:

pip install bokeh

주석

pip 명령을 쓰면 예제는 설치되지 않는다. 이 경우 예제를 보려면 Git 레포지토리를 클론해서 examples/ 디렉토리를 보아야 한다.

시작¶

보케는 많은 기능을 제공하는 대형 라이브러리이기 때문에 여기에서는 가장 흔한 사용예를 보인다. 자세한 내용은 사용자 안내서를 참조한다.

간단한 예제부터 시작하자.

기본적인 파이썬 리스트에 담긴 데이터를 확대, 이동, 저장 등의 도구가 있는 라인 플롯으로 그리는 것은 아주 간단하다:

from bokeh.plotting import figure, output_file, show

# prepare some data
x = [1, 2, 3, 4, 5]
y = [6, 7, 2, 4, 5]

# output to static HTML file
output_file("lines.html")

# create a new plot with a title and axis labels
p = figure(title="simple line example", x_axis_label='x', y_axis_label='y')

# add a line renderer with legend and line thickness
p.line(x, y, legend="Temp.", line_width=2)

# show the results
show(p)

이 스크립트를 실행하면 "lines.html"`이라는 출력 파일이 생기는 것을 볼 수 있다. 그리고 브라우저가 자동으로 새 탭을 열면서 이 파일을 보여준다. (이 문서에서는 설명을 위해 직접 플롯 출력을 보여주었다.)

bokeh.plotting 인터페이스로 플롯을 생성하는 순서는 다음과 같다.:

데이터 준비
이 코드에서는 단순한 파이썬 리스트지만 NumPy 배열이나 Pandas 시리즈도 쓸 수 있다.
출력 생성 위치 지정
이 경우에는 파일 "lines.html"로 출력하는 output_file()를 사용한다. 다른 옵션은 주피터 노트북 용 output_notebook() 명령을 쓰는 것이다.
figure() 호출
이 명령은 제목, 툴, 축 라벨 등 여러가지 커스터마이제이션도 쉽고 디폴값도 가진 플롯을 생성한다.
렌더러 추가
이 코드에서는 line() 명령을 사용하였다. 또한 색상, 범례, 폭 등을 커스터마이징 하였다.
show() 또는 save() 명령으로 결과 출력
이 함수들은 HTML 파일로 플롯을 저장하고 추가적으로 브라우저에서 보여준다.

3번쩨와 4번째 단계를 반복하면 다음 예제처럼 하나 이상의 플롯을 그릴 수 있다.

bokeh.plotting 인터페이스를 쓰면 하나 이상의 데이터 시리즈나 글리프, 로그 축 등을 쉽게 추가할 수 있다. 또한 다음 처럼 여러개의 글리프를 하나로 묶어서 보여줄 수도 있다.

from bokeh.plotting import figure, output_file, show

# prepare some data
x = [0.1, 0.5, 1.0, 1.5, 2.0, 2.5, 3.0]
y0 = [i**2 for i in x]
y1 = [10**i for i in x]
y2 = [10**(i**2) for i in x]

# output to static HTML file
output_file("log_lines.html")

# create a new plot
p = figure(
   tools="pan,box_zoom,reset,save",
   y_axis_type="log", y_range=[0.001, 10**11], title="log axis example",
   x_axis_label='sections', y_axis_label='particles'
)

# add some renderers
p.line(x, x, legend="y=x")
p.circle(x, x, legend="y=x", fill_color="white", size=8)
p.line(x, y0, legend="y=x^2", line_width=3)
p.line(x, y1, legend="y=10^x", line_color="red")
p.circle(x, y1, legend="y=10^x", fill_color="red", line_color="red", size=6)
p.line(x, y2, legend="y=10^x^2", line_color="orange", line_dash="4 4")

# show the results
show(p)

샘플 데이터¶

보케 소스에 포함된 예제 중 일부는 별도로 배포되는 샘플 데이터 파일을 사용한다. 이 데이터를 다운로드하려면 배쉬나 윈도우 명령 프롬프트에서 다음 명령을 실행한다.:

bokeh sampledata

개념¶

위 예제를 참고로하여 몇가지 핵심 개념을 정의한다.

플롯(plot)¶

플롯은 보케의 핵심개념이다. 플롯은 모든 객체(렌더러, 가이드, 데이터, 도구)를 포함하는 컨테이너로 사용자에게 보여지는 최종 시각을 조합한다. bokeh.plotting 인터페이스는 이에 필요한 모든 객체를 조합하는 것을 돕은 figure() 함수를 제공한다.

글리프(glyph)¶

글리프는 보케가 표시하는 기본 시각 표시자다. 가장 로우 레벨에서는 Line과 같은 글리프 객체가 있다. 만약 로우레벨 bokeh.models 인터페이스를 사용하면 데이터 소스와 글리프 객체 보케 객체를 생성하고 조합하는 것은 모두 사용자의 책임이다. 일을 쉽게 하기위해 bokeh.plotting 인터페이스는 첫번째 예제에 쓰인 line()와 같은 하이레벨 글리프 메소드를 제공한다. 두번제 예제에서는 원을 그리기 위한 circle() 메서드를 추가하였다. 선과 원 이외에도 다양한 glyphs와 markers를 제공한다.

글리프의 외양은 글리프 속성과 연결된 데이터 값에 연동된다. 위 예제에서 x 또는 y 위치값이 데이터 벡터로 설정되었다. 글리프는 겉모습을 제어하기 위한 Line Properties, Fill Properties, Text Properties 등의 속성도 가지고 있다. 이러한 속성들도 벡터값으로 설정할 수 있다. 다음 예제에서 살펴 볼 것이다.

가이드(guide)와 어노테이션(annotation)¶

보케는 프리젠테이션이나 비교를 돕는 시각적 요소들도 가지고 있다. 사용자가 거리, 각도 등을 볼 수 있게 돕는 시각요소를 가이드(guide)라고 한다. 그리드 라인, 밴드, 틱과 틱라벨을 가지는 (선형, 로그, 시각)축 등도 모두 가이드이다. 어노테이션은 플롯 라벨이나 각 부분의 이름을 나타내는 시각적 요소이다. 제목, 범례 등이 어노테이션에 포함된다.

레인지(range)¶

레인지는 플롯의 데이터 범위를 말한다. bokeh.plotting 인터페이스에 의해 생성되는 플롯은 디폴트로모든 데이터를 포함하도록 플롯 범위를 설정하는 DataRange1d 객체를 가지게 된다. 하지만 고정된 범위를 가지는 Range1d 객체를 설정할 수도 있다. 편의를 위해 이 값은 튜플이나 리스트로 표시할 수 있다.:

p = figure(x_range=[0,10], y_range=(10, 20))

리소스¶

플롯을 생성하기 위해 BokehJS 자바스크립트 라이브러리나 CSS 코드같은 클라이언트 라이브러리가 브라우저에 로드되어야 한다. output_file() 함수는 디폴트로 http://cdn.pydata.org 웹주소로부터 BokehJS를 로드한다. 하지만 output_file() 함수에 mode="inline" 인수를 넣어서 BokehJS 리소스를 포함하도록 임베딩된 스태틱 HTML 파일을 만들 수도 있다.

더 많은 예제¶

다음은 bokeh.plotting 인터페이스를 사용하여 많이 사용되는 업무를 처리한 예제들이다.

색상과 크기 벡터화¶

이 예제는 fill_color 또는 radius 글리프 속성으로 벡터화된 데이터 값을 넣을 수 있다는 것을 보이는 예제이다. 이 예제에서 살펴볼 점을 다음고 같다.

  • figure() 함수에 툴 이름을 리스트로 제공
  • mode 인수를 써서 BokehJS 리소스를 CDN으로 제공
  • setting the x_range and y_range explicitly
  • 선을 None 값으로 설정하여 안보이게 함
  • 데이터를 제공하는데 NumPy 배열 사용
import numpy as np

from bokeh.plotting import figure, output_file, show

# prepare some data
N = 4000
x = np.random.random(size=N) * 100
y = np.random.random(size=N) * 100
radii = np.random.random(size=N) * 1.5
colors = [
    "#%02x%02x%02x" % (int(r), int(g), 150) for r, g in zip(50+2*x, 30+2*y)
]

# output to static HTML file (with CDN resources)
output_file("color_scatter.html", title="color_scatter.py example", mode="cdn")

TOOLS = "crosshair,pan,wheel_zoom,box_zoom,reset,box_select,lasso_select"

# create a new plot with the tools above, and explicit ranges
p = figure(tools=TOOLS, x_range=(0, 100), y_range=(0, 100))

# add a circle renderer with vectorized colors and sizes
p.circle(x, y, radius=radii, fill_color=colors, fill_alpha=0.6, line_color=None)

# show the results
show(p)

상호연결된 이동 및 브러싱¶

다른 플롯의 여러가지 속성을 상호연결하는 것은 데이터 시각화에서 아주 유용하다. 보케에서느 플롯간에 몇가지 플롯 구성요소를 공유함으로써 이러한 연결을 만들 수 있다. 다음은 두 플롯간에 레인지 객체를 공유함으로써 상호연결된 이동(panning)을 구현하는 예다.:

  • figure() 함수를 여러번 호출해서 복수의 플롯을 생성
  • gridplot() 명령으로 여러개의 플롯을 배열형태로 배치
  • triangle(), square() 등의 글리프 메서드로 새로운 글리프 생성
  • toolbar_location를 None로 설정해서 툴바를 안보이게 함
  • setting convenience arguments color (sets both line_color and fill_color) and alpha (sets both line_alpha and fill_alpha)
import numpy as np

from bokeh.layouts import gridplot
from bokeh.plotting import figure, output_file, show

# prepare some data
N = 100
x = np.linspace(0, 4*np.pi, N)
y0 = np.sin(x)
y1 = np.cos(x)
y2 = np.sin(x) + np.cos(x)

# output to static HTML file
output_file("linked_panning.html")

# create a new plot
s1 = figure(width=250, plot_height=250, title=None)
s1.circle(x, y0, size=10, color="navy", alpha=0.5)

# NEW: create a new plot and share both ranges
s2 = figure(width=250, height=250, x_range=s1.x_range, y_range=s1.y_range, title=None)
s2.triangle(x, y1, size=10, color="firebrick", alpha=0.5)

# NEW: create a new plot and share only one range
s3 = figure(width=250, height=250, x_range=s1.x_range, title=None)
s3.square(x, y2, size=10, color="olive", alpha=0.5)

# NEW: put the subplots in a gridplot
p = gridplot([[s1, s2, s3]], toolbar_location=None)

# show the results
show(p)

툴바가 보이지 않지만 이동 도구는 여전히 존재하고 활성화되어 있다. 플롯을 클릭해서 드래그해보면 이동한다. 그리고 양쪽의 레인지가 연결되어 있는 것을 볼 수 있다.

또하나 유용한 상호연결은 브러시 상호연결이다. (한 플롯에서 선택하면 다른 플롯에서도 선택됨) 다음은 두 플롯간에 ColumnDataSource를 공유함으로써 브러시 상호연결을 구현한 예다.:

import numpy as np
from bokeh.plotting import *
from bokeh.models import ColumnDataSource

# prepare some date
N = 300
x = np.linspace(0, 4*np.pi, N)
y0 = np.sin(x)
y1 = np.cos(x)

# output to static HTML file
output_file("linked_brushing.html")

# NEW: create a column data source for the plots to share
source = ColumnDataSource(data=dict(x=x, y0=y0, y1=y1))

TOOLS = "pan,wheel_zoom,box_zoom,reset,save,box_select,lasso_select"

# create a new plot and add a renderer
left = figure(tools=TOOLS, width=350, height=350, title=None)
left.circle('x', 'y0', source=source)

# create another new plot and add a renderer
right = figure(tools=TOOLS, width=350, height=350, title=None)
right.circle('x', 'y1', source=source)

# put the subplots in a gridplot
p = gridplot([[left, right]])

# show the results
show(p)

박스 선택 도구나 라쏘 선택 도구를 골라서 클릭하고 드래그하여 선택을 하면 다른 플롯에서도 선택영역이 달라진다.

시간 축¶

날짜와 시간 데이터도 흔히 다루는 데이터 중의 하나이다. 보케는 현재 플롯 스케일에 따라 틱 표시를 변화시키는 DatetimeAxis 요소를 제공한다. figure()의 x_axis_type 또는 y_axis_type 인수로 "datetime" 값을 넣어서 명시적으로 시간 축을 설정할 수도 있다. 다음 예제에서 몇가지 재미있는 것을 볼 수 있다.:

  • figure()에 plot_width 및 plot_height 인수 설정
  • 플롯과 다른 객체 속성값을 정해서 커스터미이징
  • 다음 Figure 속성으로 가이드와 어노테이션 바꾸기: legend, grid, xgrid, ygrid, axis, xaxis, yaxis
import numpy as np

from bokeh.plotting import figure, output_file, show
from bokeh.sampledata.stocks import AAPL

# prepare some data
aapl = np.array(AAPL['adj_close'])
aapl_dates = np.array(AAPL['date'], dtype=np.datetime64)

window_size = 30
window = np.ones(window_size)/float(window_size)
aapl_avg = np.convolve(aapl, window, 'same')

# output to static HTML file
output_file("stocks.html", title="stocks.py example")

# create a new plot with a a datetime axis type
p = figure(plot_width=800, plot_height=350, x_axis_type="datetime")

# add renderers
p.circle(aapl_dates, aapl, size=4, color='darkgrey', alpha=0.2, legend='close')
p.line(aapl_dates, aapl_avg, color='navy', legend='avg')

# NEW: customize by setting attributes
p.title.text = "AAPL One-Month Average"
p.legend.location = "top_left"
p.grid.grid_line_alpha = 0
p.xaxis.axis_label = 'Date'
p.yaxis.axis_label = 'Price'
p.ygrid.band_fill_color = "olive"
p.ygrid.band_fill_alpha = 0.1

# show the results
show(p)

Bokeh also comes with an optional server component, the Bokeh Server. It is possible to create many interesting and interactive visualizations without using the Bokeh server, as we have seen above. However, the Bokeh server affords many novel and powerful capabilities, including:

  • Intelligent server-side downsampling of large datasets.
  • Streaming data automatically updating plots.
  • Sophisticated glyph re-writing and transformations for "Big Data".
  • Plot and dashboard publishing for wider audiences.
